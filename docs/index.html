⚡ Quick Start

From zero to running API in under a minute.

Create a new LatticePHP project:

$ composer create-project lattice/starter-api myapp
Terminal
# Create project
composer create-project lattice/starter-api myapp
cd myapp

# Configure environment (SQLite by default)
cp .env.example .env

# Run migrations and seed data
php lattice migrate
php lattice db:seed

# Start development server
php lattice serve
# Server running on http://localhost:8000

# Verify it works
curl http://localhost:8000/api/health
# {"status":"ok"}

# Run the test suite
php lattice test

Scaffold a module

Terminal
php lattice make:module Contacts --crud

# Creates:

📌 Features

Everything you need to build production-grade APIs and services.

📦

Module System

NestJS-style modules with explicit imports, exports, providers, and controllers. Every dependency is visible in code. No hidden globals.

🎯

Attribute Routing

#[Controller], #[Get], #[Post] — routes are declared on controllers and methods. Zero config files.

🛡

Guards & Pipeline

Guards, pipes, interceptors, and exception filters that compose cleanly and work across HTTP, gRPC, and message transports.

⛰

Durable Workflows

Temporal-class workflow orchestration built on your existing database and queue. Deterministic replay, signals, queries, and sagas.

🔒

Authentication

JWT (access + refresh), personal access tokens, API keys, OAuth2/OIDC server, and headless social auth. All pluggable.

📋

Validated DTOs

#[Body] CreateDto $dto — auto-deserializes request body, validates with illuminate/validation, returns 422 on failure.

🗃

Eloquent Built-In

Full Eloquent ORM, query builder, migrations, seeders, and factories. Same API you know from Laravel — no wrappers.

🏢

Workspaces

#[BelongsToWorkspace] auto-scopes all queries. Built-in invitations, member roles, and workspace switching.

📑

OpenAPI Generation

First-class OpenAPI spec generation from your route metadata, DTOs, and response types. Always in sync with your code.

🔍

Observability

Structured logging, OpenTelemetry distributed tracing, Prometheus metrics, and health checks. Production-ready from day one.

🚀

Microservices

Transport-aware controllers with pluggable adapters: NATS, RabbitMQ, SQS, Kafka, and gRPC. Same code, different transport.

🔧

CQRS

CommandBus and QueryBus with #[CommandHandler] / #[QueryHandler]. Async command dispatch via queue.

🧱 Architecture

Four layers. Clear boundaries. Dependencies only flow downward.

app/Modules/Contacts/ContactController.php
#[Controller('/api/contacts')]
#[UseGuards(guards: [JwtAuthenticationGuard::class, WorkspaceGuard::class])]
final class ContactController
{
    public function __construct(
        private readonly ContactService $service,
    ) {}

    #[Get('/')]
    public function index(Request $request): Response
    {
        $filter = QueryFilter::fromRequest($request->query);
        $contacts = Contact::filter($filter)
            ->with(['company'])
            ->paginate($filter->getPerPage());

        return ResponseFactory::paginated($contacts, ContactResource::class);
    }

    #[Post('/')]
    public function store(
        #[Body] CreateContactDto $dto,
        #[CurrentUser] Principal $user,
    ): Response {
        $contact = $this->service->create($dto, $user);
        Log::info('Contact created', ['id' => $contact->id]);
        return ResponseFactory::created(
            ['data' => ContactResource::make($contact)->toArray()]
        );
    }

    #[Get('/:id')]
    public function show(#[Param] int $id): Response
    {
        $contact = Contact::with(['company', 'deals', 'activities'])->findOrFail($id);
        return ResponseFactory::ok(['data' => ContactResource::make($contact)->toArray()]);
    }
}

Key patterns used: Module system, attribute routing, JWT + workspace guards, DTO validation, Eloquent with query filters, API resources, structured logging, and full test coverage.

⛰ Durable Workflows

Temporal-class orchestration. No external services. Your database + queue.

OrderFulfillmentWorkflow.php
#[Workflow]
final class OrderFulfillmentWorkflow
{
    private string $status = 'pending';

    public function execute(WorkflowContext $ctx, array $input): array
    {
        // Each activity retries independently. State survives crashes.
        $payment = $ctx->executeActivity(
            PaymentActivity::class, 'charge',
            [$input['amount']],
            options: new ActivityOptions(
                retryPolicy: new RetryPolicy(maxAttempts: 3),
            ),
        );

        $shipping = $ctx->executeActivity(
            ShippingActivity::class, 'ship',
            [$input['address']],
        );

        $this->status = 'fulfilled';
        return ['payment' => $payment, 'shipping' => $shipping];
    }

    #[SignalMethod]
    public function cancel(): void
    {
        $this->status = 'cancelled';
    }

    #[QueryMethod]
    public function getStatus(): string
    {
        return $this->status;
    }
}
